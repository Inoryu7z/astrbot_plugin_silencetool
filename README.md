# 🔇 SilenceTool · 静默工具

在工具调用时抑制 LLM 的中间文本输出，避免消息轰炸。

---

## 🤔 这是什么问题？

当 LLM 决定调用工具时，它通常会同时返回一段文本（如"好，我马上查询"）和工具调用指令。AstrBot 会先把这段文本发给用户，再执行工具，最后把工具结果也发出去。

结果是用户会收到两条消息：

1. "好，我马上查询" ← 中间文本（工具调用前）
2. "今天北京 25℃，晴天。" ← 最终结果（工具调用后）

在角色扮演等场景下，第一条消息往往是多余的，甚至破坏沉浸感。理想情况是只收到最终结果。

本插件的作用就是**自动抑制第一条中间文本**，只保留最终结果消息。

---

## 🆚 和 AstrBot 自带功能的区别

AstrBot 提供了两个相关配置项，但都不能完全解决这个问题：

| 功能 | 效果 | 和本插件的区别 |
|------|------|---------------|
| `buffer_intermediate_messages` | 把中间文本缓冲，最后**合并成一条**发送 | 用户仍会看到中间文本，只是和最终结果拼在一起了。本插件是**完全不发送**中间文本 |
| `show_tool_use_status` | 控制"🔨 调用工具: xxx"状态消息 | 这控制的是框架生成的状态提示，不是 LLM 自己说的"好，我马上查询"。两者是不同的东西 |

简单说：
- `buffer_intermediate_messages` = 合并发送（还是会看到）
- `show_tool_use_status` = 控制框架提示（和 LLM 文本无关）
- **本插件** = 完全不发送中间文本（只保留最终结果）

---

## ✨ 功能

- 自动抑制工具调用前 LLM 生成的中间文本
- 只保留工具调用完成后的最终结果消息
- 支持群聊/私聊独立开关

---

## 🛠️ 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `enable_silence` | 是否启用静默 | `true` |
| `enable_group_process` | 是否处理群聊 | `true` |

---

## ⚠️ 注意事项

1. **仅非流式模式有效**：流式输出（`streaming_response=True`）下，中间文本通过 streaming delta 直接发送，框架没有提供拦截点，本插件无法生效。
2. **依赖内部 API**：本插件使用 AstrBot 内部字典 `_ACTIVE_AGENT_RUNNERS` 判断 Agent 是否仍在运行。这是下划线开头的私有 API，未来 AstrBot 版本更新可能需要适配。
3. **不影响工具执行**：本插件只抑制文本发送，不会阻止工具调用本身。工具照常执行，只是用户不会看到"好，我马上查询"这类中间文本。
4. **不影响最终结果**：工具调用完成后的最终 LLM 回复会正常发送，不受影响。
